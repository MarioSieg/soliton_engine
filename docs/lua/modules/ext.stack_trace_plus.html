stacktrace (thread, message, level)
Public: Collects a detailed stack trace, dumping locals, resolving function names when they're not available, etc. This function is suitable to be used as an error handler with pcall or xpcall

Parameters:

  • thread An optional thread whose stack is to be inspected (defaul is the current thread)
  • message An optional error string or object.
  • level An optional number telling at which level to start the traceback (default is 1)

    Returns a string with the stack trace and a string with the original error.
