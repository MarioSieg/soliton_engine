◆ call()

template<class... Args>
LuaResult luabridge::call ( const LuaRef & object,
Args &&... args )

Safely call Lua code.

These overloads allow Lua code to be called throught lua_pcall. The return value is provided as a LuaResult which will hold the return values or an error if the call failed.

If an error occurs, a LuaException is thrown or if exceptions are disabled the FunctionResult will contain a error code and evaluate false.

Note
The function might throw a LuaException if the application is compiled with exceptions on and they are enabled globally by calling enableExceptions in two cases:
  • one of the arguments passed cannot be pushed in the stack, for example a unregistered C++ class
  • the lua invaction calls the panic handler, which is converted to a C++ exception
Returns
A result object.
